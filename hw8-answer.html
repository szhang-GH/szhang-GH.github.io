  1. Multicollinearity and Variance Inflation Factor (VIF). Load the built-in mtcars dataset in R using data(mtcars). We want to build a multiple linear regression model predicting mpg (Miles/(US) gallon) using disp (Displacement), hp (Gross horsepower), wt (Weight), and cyl (Number of cylinders).

    1. Fit a full model (model1) predicting mpg using disp, hp, wt, and cyl. Print the summary. Which, if any, predictors are statistically significant at the \alpha = 0.05 level?
    2. Use the vif() function from the car package to calculate the Variance Inflation Factor for each predictor in model1. Which predictor(s) show strong signs of multicollinearity (e.g., VIF > 10)?
    3. Verify the VIF for disp manually. Fit an auxiliary regression model predicting disp using the other three predictors (hp, wt, cyl). Extract the unadjusted R^2 of this model and compute \frac{1}{1-R^2}. Does it match the VIF from part (b)?
    4. To mitigate multicollinearity, drop the variable with the highest VIF (disp) and fit a new model (model2) predicting mpg using only hp, wt, and cyl. Calculate the VIFs for this new model. Did removing disp reduce the standard errors for the coefficients of the remaining variables?

    Answer:

    1. Full Model Fit.

      ::: {.cell}

      model1 <- lm(mpg ~ disp + hp + wt + cyl, data = mtcars)
summary(model1)$coefficients

      ::: {.cell-output .cell-output-stdout}

disp         0.01159924 0.01172681  0.9891215 3.313856e-01

wt          -3.85390352 1.01547364 -3.7951783 7.589470e-04
cyl         -1.29331972 0.65587675 -1.9718944 5.894681e-02

      ::: :::

      At the \alpha = 0.05 level, only wt is statistically significant (p < 0.001). The other predictors are not significant.

    2. VIF Calculation.

      ::: {.cell}

      library(car)
vif(model1)

      ::: {.cell-output .cell-output-stdout}

10.373286  3.405983  4.848016  6.737707

      ::: :::

      The variable disp shows a clear sign of severe multicollinearity with a VIF of 10.37 (which exceeds the common threshold of 10). cyl also has a moderately high VIF of 6.74.

    3. Manual VIF Verification.

      ::: {.cell}

      aux_model <- lm(disp ~ hp + wt + cyl, data = mtcars)
r2_disp <- summary(aux_model)$r.squared
vif_disp_manual <- 1 / (1 - r2_disp)
vif_disp_manual

      ::: {.cell-output .cell-output-stdout}

      ::: :::

      Yes, the manual calculation (10.37) exactly matches the VIF for disp from the car package.

    4. Refitting Model without disp.

      :::

      The VIFs for all remaining variables decreased substantially (all are now below 5). Removing the highly collinear variable reduced the variance in our estimates, and they became much more stable.

  2. Structural Multicollinearity and Centering. Using the mtcars dataset, suppose we want to model the non-linear relationship between mpg and hp (horsepower) using a quadratic polynomial model.

    1. Fit a polynomial regression model (poly_model) predicting mpg using hp and its squared term I(hp^2). Calculate the VIFs for this model using vif(). What do you notice?
    2. This is an example of structural multicollinearity (multicollinearity that is a mathematical artifact caused by creating new predictors from existing ones, such as squaring a term). To mitigate it, create a new centered variable hp_c by subtracting the mean of hp from each hp value (mtcars$hp_c <- mtcars$hp - mean(mtcars$hp)).
    3. Fit a new polynomial model (poly_model_c) using hp_c and I(hp_c^2). Recalculate the VIFs. Did centering resolve the structural multicollinearity issue?
    4. Compare the Adjusted R^2 of poly_model and poly_model_c. Are they different?

    Answer:

    1. Polynomial Model without Centering.

      ::: {.cell}

      poly_model <- lm(mpg ~ hp + I(hp^2), data = mtcars)
vif(poly_model)

      ::: {.cell-output .cell-output-stdout}

            hp  I(hp^2) 
18.72562 18.72562

      ::: :::

      The VIF values are extremely high (roughly 18.7), indicating severe structural multicollinearity between hp and hp^2.

    2. Creating Centered Variable.

      ::: {.cell}

      mtcars$hp_c <- mtcars$hp - mean(mtcars$hp)

      :::

    3. Polynomial Model with Centering.

      ::: {.cell}

      poly_model_c <- lm(mpg ~ hp_c + I(hp_c^2), data = mtcars)
vif(poly_model_c)

      ::: {.cell-output .cell-output-stdout}

           hp_c I(hp_c^2) 
 1.393755  1.393755

      ::: :::

      Yes, centering drastically reduced the VIF values down to 1.39. The structural multicollinearity issue is now completely resolved.

    4. Comparing Adjusted R^2.

      ::: {.cell}

      summary(poly_model)$adj.r.squared

      ::: {.cell-output .cell-output-stdout}

      [1] 0.73929

      :::

      summary(poly_model_c)$adj.r.squared

      ::: {.cell-output .cell-output-stdout}

      [1] 0.73929

      ::: :::

      The Adjusted R^2 values are identical (0.739). Centering mitigates structural multicollinearity and stabilizes our coefficient estimates, but it does not change the model’s overall fit or its predictive explanatory power.

      Extra learning: Unlike inherent multicollinearity from observational data, structural multicollinearity is a mathematical artifact created by the modeler when creating new predictors from existing ones. Common examples include polynomial terms (e.g., including both x and x^2) and interaction terms (e.g., including x_1, x_2, and x_1 \times x_2). As demonstrated above, the best solution is to mean-center these variables (e.g., x^* = x - \bar{x}) before creating polynomial or interaction terms in order to strongly mitigate the structural multicollinearity.

  3. Model Selection Criteria (Adjusted R^2, Mallows’ C_p, AIC, BIC). Load the built-in swiss dataset using data(swiss). This dataset contains a standardized fertility measure and 5 socio-economic indicators for 47 French-speaking provinces of Switzerland. We want to evaluate three different models predicting Fertility.

    Fit the following three models:

    • Model A: Fertility ~ Education
    • Model B: Fertility ~ Education + Agriculture + Catholic
    • Model C (Full Model): Fertility ~ . (uses all 5 available predictors)
    1. Extract the Adjusted R^2 for each of the three models. Which model is preferred based on Adjusted R^2?
    2. Use the AIC() function to calculate the AIC for each of the three models. Which model is preferred based on AIC?
    3. Use the BIC() function to calculate the BIC for each of the three models. Which model is preferred based on BIC?
    4. Calculate Mallows’ C_p for Model B. Use the full model (Model C) to estimate the true error variance \sigma^2. Remember the formula: C_p = \frac{\text{SSE}}{\sigma^2} - n + 2p. Here p is the number of parameters in the evaluated model (including the intercept), and n is the sample size. Is Model B’s C_p close to p?

    Answer:

    # Load data and fit models for subsequent answers
data(swiss)
modelA <- lm(Fertility ~ Education, data = swiss)
modelB <- lm(Fertility ~ Education + Agriculture + Catholic, data = swiss)
modelC <- lm(Fertility ~ ., data = swiss)

    1. Adjusted R^2.

      ::: {.cell}

      c(ModelA = summary(modelA)$adj.r.squared, 
  ModelB = summary(modelB)$adj.r.squared, 
  ModelC = summary(modelC)$adj.r.squared)

      ::: {.cell-output .cell-output-stdout}

         ModelA    ModelB    ModelC 
0.4281849 0.6172951 0.6709710

      ::: :::

      Based on Adjusted R^2, the highest value is preferred, which corresponds to the full model, Model C (0.671).

    2. AIC.

      ::: {.cell}

      AIC(modelA, modelB, modelC)

      ::: {.cell-output .cell-output-stdout}

             df      AIC
modelA  3 348.4223
modelB  5 331.4126
modelC  7 326.0716

      ::: :::

      Based on the AIC (where a lower score is better), the preferred model is Model C, which has the lowest AIC of 326.07.

    3. BIC.

      ::: {.cell}

      BIC(modelA, modelB, modelC)

      ::: {.cell-output .cell-output-stdout}

             df      BIC
modelA  3 353.9727
modelB  5 340.6634
modelC  7 339.0226

      ::: :::

      Based on the BIC (where a lower score corresponds to a better model and higher parameter penalties apply), the preferred model is Model C (339.02), though it is relatively close to Model B (340.66).

    4. Mallows’ C_p.

      ::: {.cell}

      sigma2_hat <- summary(modelC)$sigma^2
sse_B <- sum(residuals(modelB)^2)
n <- nrow(swiss)
p <- length(coef(modelB)) # Intercept + 3 predictors = 4 parameters

Cp_B <- sse_B / sigma2_hat - n + 2 * p
Cp_B

      ::: {.cell-output .cell-output-stdout}

      [1] 11.01477

      ::: :::

      For Model B, Mallows’ C_p is 11.01. This value is notably larger than p = 4. Because C_p \gg p, it suggests that Model B is biased and does not capture all the systematic variation modeled by the full model.
